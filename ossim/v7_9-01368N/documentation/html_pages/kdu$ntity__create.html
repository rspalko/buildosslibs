kdu_thread_entity::create

Java: Kdu_thread_entity.Create

void create( kdu_int64 cpu_affinity=0, bool also_set_owner_affinity=true)

Java: public native void Create( )

Go to class description.

Synopsis

You must call this function after construction to prepare the thread entity for use. Prior to this point exists will return false. After creation, the present object will become the owner of a thread group, to which workers may be added using the add_thread function.

Arguments

cpu_affinity [kdu_int64]

The functionality of this argument has been superceded by the more powerful set_cpu_affinity function, so you are recommended to pass 0 for this argument and use that function instead.

If you do supply a non-zero value for this argument, it is passed as the affinity_mask argument in calls to kdu_thread::set_cpu_affinity that are issued whenever a new worker thread is added via the add_thread function. The affinity_context argument to all such calls will be 0, which potentially limits the set of CPUs on which threads can be scheduled.

also_set_owner_affinity [bool]

If true, and cpu_affinity is non-zero, the cpu_affinity argument is also used to set the CPU affinity of the calling thread (the thread group owner), again passing an affinity_context value of 0 in the call to kdu_thread::set_cpu_affinity. Otherwise, this function makes no attempt to change the calling thread's CPU affinity.

As with the cpu_affinity argument, from KDU-7.5 onwards, you are recommended to use the more powerful set_cpu_affinity function rather than doing so during the create call.

It is also worth noting that the calling thread can always set its own CPU affinity at any time by explicitly invoking kdu_thread::set_cpu_affinity on a kdu_thread object initialized via kdu_thread::set_to_self.
