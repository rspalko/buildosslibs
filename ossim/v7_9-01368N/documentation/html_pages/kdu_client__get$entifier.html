kdu_client::get_cache_identifier

Java: Kdu_client.Get_cache_identifier

const char * get_cache_identifier( )

Java: public native String Get_cache_identifier( )

Go to class description.

Synopsis

This function is provided to allow a client to determine whether or not the logical resource that it is working with is compatible with that represented by a separate cache file. To do this, you pass the separate named cache file to check_cache_file, passing it a kdu_cache_file_info object, then you compare the string returned by kdu_cache_file_info::get_cache_identifier with that returned by this function. This function can also be used to determine whether two separate kdu_client objects are both working with the same logical resource/target.

This function cannot return non-NULL unless target_started already returns true, which means that the first reply has been received in response to a connect call or there has been a successful open_with_cache_file call. Once this has happened, the cache identifier can usually be completed based on the target_id string and then this function can return non-NULL. Note, however, that if the server never returns a usable target-id (something other than 0) this function will always return NULL.

If non-NULL, the returned string is an internal resource that should remain valid until close is called (perhaps implicitly from within a call to connect or open_with_cache_file).
