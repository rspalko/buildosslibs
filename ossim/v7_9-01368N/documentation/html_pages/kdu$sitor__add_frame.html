kdu_region_compositor::add_frame

Java: Kdu_region_compositor.Add_frame

bool add_frame( jpx_frame_expander * expander, kdu_coords offset)

Java: public native boolean Add_frame( Jpx_frame_expander expander, Kdu_coords offset)

Go to class description.

Synopsis

You can call this function one or more times after set_frame to build up compositions that include the content from multiple original JPX composited frames. Usually, the expander passed to this function would be different to that passed to set_frame, or would at least have been reconfigured, so that it represents a different JPX source frame. However, this is not required. Also, offset will usually be different from that passed to set_frame so that the added frames can sit side-by-side, or within a grid, but any amount of overlap can be introduced between frames if you like — their Z-order follows the order in which they are added.

This function is particularly useful for mobile apps, where content is usually rendered to a fixed size window (full display). In this case, if the content is rendered at a very small scale, it is often intuitive to lay out multiple frames on a grid and display them as a single image — sometimes called a "contact sheet". The function can also be used to implement interactive "page flipping", where consecutive frames are interpreted as pages, and certain types of user navigation slide the viewport between the pages.

Regardless of the application, the mechanism is completely general.

Return Value

False if any of the compositing layers (members) found within expander is not yet fully available. In this case, the function still augments the compositing surface, but may not add some compositing layers which will eventually belong to that surface. Indeed even if expander contains no members at all, the surface is still augmented to reflect the region that the frame will eventually occupy.

Arguments

expander [jpx_frame_expander *]

Unlike set_frame, the epander argument passed to this function can be NULL or refer to an object that has an incomplete (or even empty) set of members. This can be useful, since it allows space to be reserved within the composition for an incomplete or non-existent frame.

offset [kdu_coords]

Interprted as a displacement of the original image layers, after application of all relevant compositing instructions, as found within expander. Each frame of a JPX source has exactly the same original image dimensions, notionally anchored at (0,0), but this function constructs the overall composition that is formed from a union of such frame rectangles, each displaced by its offset. Morever, offset may be negative, so the new overall composition region is generally larger than that of the nominal JPX composition frame that need no longer be anchored at (0,0).
