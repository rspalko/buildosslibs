Synopsis

NOTE: This function is probably only useful to you if you are using the new_rendered_region that is produced by calls to process to incrementally paint composited data to a display. If you are using the kdu_compositor_buf::get_rendering_status method instead to determine what content should be painted to a display (generally the preferred approach), calls to this function will not generally result in display updates in the invalidated region, so it is best to avoid its use. With this in mind, it may be a good idea to consider this function deprecated, although we do not have any immediate intention of deprecating it.

Causes invalidated_region to be added to the internal machinery which keeps track of rectangular regions which need to be included as newly_rendered_region results from calls to process, before is_processing_complete can return true. The explicit invalidation of regions at most causes them to be recomposited onto a compositing surface — this is quite different to refresh, which results in complete re-rendering of the content.

The function is provided for the benefit of applications which need to repaint some portion of a composition to their display buffer, but want the repainting to be driven by returns from the process function so that a region whose underlying imagery has yet to be decompressed and rendered will not be repainted prematurely.

Like process, this function interprets invalidated_region with respect to the rendering scale, which is the product of the scale and rendering_scale_adjustment arguments passed in the last call to set_scale. All other API functions offered by the region-compositor interpret dimensions and locations of their arguments and return values with respect only to the scale parameter passed to set_scale.
