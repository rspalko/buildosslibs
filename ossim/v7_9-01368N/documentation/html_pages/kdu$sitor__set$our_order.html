kdu_region_compositor::set_colour_order

Java: Kdu_region_compositor.Set_colour_order

void set_colour_order( kdu_colour_order colour_order)

Java: public native void Set_colour_order( int colour_order)

Go to class description.

Synopsis

This function is normally called right after create to configure the colour channel ordering convention used by all buffers. It is not advisable to change the colour ordering convention after rendering has commenced, since this may lead to an unpredictable mixture of content rendered with different conventions.

All surface buffers hold four channels per pixel, the first of which is always Alpha, followed either by Red then Green then Blue, or by Blue then Green then Red, depending on the colour_order value.

  • If colour_order is KDU_ARGB_ORDER, floating point buffers interleave the channels in the order Alpha, Red, Green and then Blue, while 32-bit pixel buffers record Alpha in the most significant byte, Red in the next most significant byte, then Green and finally Blue in the least significant byte.
  • If colour_order is KDU_ABGR_ORDER, floating point buffers interleave the channels in the order Alpha, Blue, Green and then Red, while 32-bit pixel buffers record Alpha in the most significant byte, Blue in the next most significant byte, then Green and finally Red in the least significant byte.

The default colour order (if this function is never called) is KDU_ARGB_ORDER.

colour_order [kdu_colour_order]
