kdu_client::augment_with_cache_file

Java: Kdu_client.Augment_with_cache_file

virtual bool augment_with_cache_file( const char * filename)

Java: public native boolean Augment_with_cache_file( String filename)

Go to class description.

Synopsis

This function is similar to open_with_cache_file, except that it can only succeed when invoked on a kdu_client object that is already open (via connect or open_with_cache_file) and is working with the same logical resource as that described by the cache file named by filename. Typically, you would not call this function without first calling the static check_cache_file, passing it a kdu_cache_file_info object and comparing the string returned by kdu_cache_file_info::get_cache_identifier with that returned by this object's get_cache_identifier function. If both the comparision reveals identical cache-identifiers, this function should succeed, returning true.

What the function actually does is augment the object's cache with the content found within the supplied cache file. This might make no difference at all, especially if the supplied cache file is one that was originally passed to the current object's open_with_cache_file function. If any data-bin is actually augmented by the call, and the current object is currently connected to a JPIP server, future requests to the server will be modified (if relevant) to identify the agumented content in JPIP cache model statements, so that it is not redundantly communicated.

If the function fails, returning false, there should be no adverse impact on the current client or any of its dependencies.

Arguments

filename [const char *]
