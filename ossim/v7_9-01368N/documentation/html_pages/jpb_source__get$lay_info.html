Synopsis

Returns information about the mastering display device associated with the currently open image. If no image is currently open, the function returns information about the next image which will be opened by open_image. If there is no mastering display information available for the image in question (currently open image or next one to be opened if none are open), the function returns false, but sets the argument values to reflect the default mastering display parameters defined in IS15444-1/AMD8, which have in turn been taken from ITU recommendation BT.709-5.

The arguments identify the chromaticities of three primaries, the whitepoint, and the minimum and maximum luminance of the display in candelas per square metre. The documentation in IS15444-1/AMD8 suggests that the order in which primaries should be recorded is green, blue and then red, but this is not something you should rely upon. In fact, the default parameters supplied in the standard even appear in a different order (red, blue then green).

Return Value

Returns true if the currently open frame has a dmon box (corresponds to jpb_mastering_display_4cc), or there is a next frame that would be opened by the next call to open_image and that frame has a dmon box. Otherwise the function returns false. However, in both cases, the values returned via the function's arguments are meaningful.
