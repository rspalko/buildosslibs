kdu_client::construct_jpip_url

Java: Kdu_client.Construct_jpip_url

virtual char * construct_jpip_url( )

Java: public native String Construct_jpip_url( )

Go to class description.

Synopsis

This function returns a string that has been allocated using the global C++ vector new[] function; it is the caller's responsibility to delete the string using delete[], unless it is NULL.

The objective is to return a full JPIP URL that could potentially be used with connect, regardless of whether the connect function or the open_with_cache function were used to get to this point. In practice, you rarely if ever need to rely upon this mechanism to generate a URL that would be passed to connect, since the reconnect function is a much better choice. However, the returned URL can be useful in constructing an informative display name for the active JPIP client, in a manner that is independentt of whether a connection was actually established using connect or the results of a previous connection were re-instated using open_with_cache_file.

If the client is not currently active (i.e., no call to connect or open_with_cache_file since construction or the last call to close), the function returns NULL.
