kdu_event::protected_set

bool protected_set( )

Go to class description.

Synopsis

Set the synchronization object to the signalled state. This causes any waiting thread to wake up and return from its call to wait. Any future call to wait by this or any other thread will also return immediately, until such point as reset is called. If the create function was supplied with manual_reset = false, the condition created by this present function is reset as soon as any single thread returns from a current or future wait call. See create for more on this.

It is not generally safe to call this function from a context in which the mutex used in calls to wait is not locked by the caller. For situations where this is difficult or impossible to arrange, you should use unprotected_set, but be sure to read the discussion of these two functions in the preamble to the kdu_event class documentation.

Return Value

False if the event has not been successfully created.
