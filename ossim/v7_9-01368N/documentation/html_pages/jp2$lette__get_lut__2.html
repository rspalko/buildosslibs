jp2_palette::get_lut

Overload navigation: 1, 2

bool get_lut( int lut_idx, kdu_sample16 lut[], int data_format=JP2_CHANNEL_FORMAT_DEFAULT, int format_param=0)

Go to class description.

Synopsis

Offsets and normalizes the original information for the indicated palette lookup table, writing the result to the supplied lut array. The lut entries are filled out as signed, normalized quantities in the range -2^{KDU_FIX_POINT-1} to +2^{KDU_FIX_POINT-1}. The selection of this particular representation is intended to facilitate interaction with the sample data processing operations conducted by the objects defined in "kdu_sample_processing.h". See kdu_line_buf and kdu_convert_ycc_to_rgb for example.

Return Value

True unless the arguments are invalid.

Arguments

lut_idx [int]

Must be in the range 0 through num_luts-1, where num_luts is the value returned by get_num_luts. If not, the function does nothing, returning false.

lut[] [kdu_sample16]

Regardless of whether get_signed returns true or false, the entries in this array are filled out with signed data, normalized to the range -2^{KDU_FIX_POINT-1} to +2^{KDU_FIX_POINT-1}. The normalization is such that the lower bound here corresponds to an actual LUT value of -2^{B-1} and the upper bound corresponds to an actual LUT value of +2^{B-1}, after subtracting 2^{B-1} from originally unsigned palette entries, where B is the bit-depth returned by get_bit_depth. The upper end of this range is, of course, not quite attainable, since the maximum B-bit integer is 2^{B-1}-1 after level adjustment of unsigned quantities.

Note that for data_format values other than 0 (i.e., JP2_CHANNEL_FORMAT_DEFAULT), the upper end of the range is generally attainable — below for more on this.

data_format [int]

See the first form of get_lut for an explanation of this argument. All the same conversions are applied to non-default formatted palette values by this function, except that level adjusted, scaled real values are then converted to 16-bit fixed point quantities by multiplying them by 2^ KDU_FIX_POINT and rounding to the nearest integer, clipping the results only to ensure that they can be accommodated by the 16-bit representation. In practice, with the typical value of KDU_FIX_POINT=13, this means that the available dynamic range is roughly from -4.0 to +4.0.

The fixed-point representation is not recommended for high dynamic range data, which might be intended to exhibit values much larger than the nominal maximum intensity in regions of intense luminosity.

format_param [int]
