nlt_params::apply_fwd_gamma

Go to class description.

Synopsis

This function can be used to apply the gamma function associated with an NLTgamma attribute to an array of non-negative input values that are assumed to be normalized, so that 1.0 corresponds to the nominal maximum intensity, while 0.0 corresponds to the absence of any light. Normally, the function would be used to construct a lookup table, which is why the array of values is named lut. The gamma array should have at least 5 entries, since it is used to supply the E, S, T, A and B parameters of the NLTgamma attribute (in that order).

The values found in lut on entry are transformed according to the forward gamma operator, that maps original image samples z to the values y that are subject to linear transformation and coding during compression. For decompression, the apply_rev_gamma function should be used.
