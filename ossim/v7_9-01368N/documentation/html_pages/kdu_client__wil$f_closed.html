kdu_client::will_lose_data_if_closed

Java: Kdu_client.Will_lose_data_if_closed

bool will_lose_data_if_closed( )

Java: public native boolean Will_lose_data_if_closed( )

Go to class description.

Synopsis

The role of this function is similar to that of get_cache_file_handling, but the return value describes what will happen to any content that may currently be in the cache if close is called, rather than describing cache file manipulations that are intended. For example, if the get_cache_file_handling function returns 0, there are no scheduled deletes or saves of a cache file, but data will only be lost if there have been updates to the cache since the last time any cache file was loaded.

One reason for providing this function is that it allows memory-challenged applications to determine which kdu_client objects can be safely closed and later re-opened without losing information recovered from a remote server — recovering such information again would incur a communications cost.
