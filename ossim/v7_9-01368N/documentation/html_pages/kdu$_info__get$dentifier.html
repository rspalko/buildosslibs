kdu_cache_file_info::get_cache_identifier

Java: Kdu_cache_file_info.Get_cache_identifier

const char * get_cache_identifier( )

Java: public native String Get_cache_identifier( )

Go to class description.

Synopsis

Returns NULL if the object has not been initialized by a successful call to kdu_client::check_cache_file or has been reset since. Otherwise, the function returns a special "cache-identifier" string, which can be compared with that returned by an instantiated kdu_client object's kdu_client::get_cache_identifier function, as a precursor to either joining with that client object or opening a new one.

The returned string contains the server-assigned target-id as its suffix, preceded immediately by a punctuation character (typically a dash '-'), which is then preceded by a target name that is derived either from the resource or the explicit target name passed in the original JPIP request, whichever is more informative, and including any sub-target information.
