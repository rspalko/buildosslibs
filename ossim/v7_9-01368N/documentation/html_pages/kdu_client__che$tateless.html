kdu_client::check_stateless

Java: Kdu_client.Check_stateless

bool check_stateless( )

Java: public native boolean Check_stateless( )

Go to class description.

Synopsis

You can use this function to determine whether or not the client is currently connected in a stateless session with a server. Calls to connect or reconnect do not immediately change the value returned by this function to true, even if a stateful session transport is requested from the server. Instead, the stateless condition persists until connection is established and the target_started function returns true. At that point, the function will continue to return false only if no stateful session was granted. Calls to close or open_with_cache_file also leave the object in the stateless condition.

There is no fundamental reason why an application should need to know whether communication is stateful or stateless, but stateful communication is significantly more efficient, so this information may allow an application to determine whether the resource should be accessed from a different server that might support stateful communication.
