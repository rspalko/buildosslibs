kdu_client::check_cache_file

Java: public native boolean Check_cache_file( String filename, Kdu_cache_file_info info)

Go to class description.

Synopsis

A call to this static function is a useful precurser to calling open_with_cache_file, since that function either succeeds or else generates an error through kdu_error.

This function generally needs to open the file at filename and read its header, after which the file is closed before the function returns.

If the check succeeds, returning true, any non-NULL info object is initialized to provide valuable information derived from the cache file's header. This information is intended to be completely consistent with information that could be retrieved from a live kdu_client object that has been opened using the cache file ( kdu_client::open_with_cache_file), or that may have saved its cache contents to the cache file.

Most important is the "cache-identifier" that is supposed to uniquely identify the resource/target represented by the cache file. This string can be retrieved via info→get_cache_identifier and then directly compared with the string returned by an instantiated kdu_client object's get_cache_identifier function to determine whether the instantiated client is already working with the same logical target. If so, there may be no need for an application to construct a new instance of kdu_client; instead, it is preferable to simply attach (see kdu_cache::attach_to) to the existing client, potentially after augmenting its cache with the contents of the file at filename — see kdu_client::augment_with_cache_file.

Return Value

True if the cache file can be opened and its header can be understood.

Arguments

filename [const char *]

info [kdu_cache_file_info *]

This argument may be NULL if you only want to discover whether the cache file exists and can be interpreted. However, you will usually want to provide a reference to a kdu_cache_file_info object that you have instantiated here. If the function returns false, no change will be made to the info object. Otherwise, it will be freshly initiated with information recovered from the cache file header.
