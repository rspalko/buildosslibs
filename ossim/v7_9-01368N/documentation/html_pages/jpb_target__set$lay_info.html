Synopsis

Calls to this function are optional, allowing you to define the colorimetry of the mastering display, to be recorded in the metadata of each frame via the optional dmon box (identified by jpb_mastering_display_4cc). The arguments identify the chromaticities of three primaries, the whitepoint, and the minimum and maximum luminance of the display in candelas per square metre. The documentation in IS15444-1/AMD8 suggests that the order of the primaries be green, blue then red.

If this function is not called, no dmon box is written. Each call to the function affects the contents of the dmon box written to the frame opened by the next call to close_image. If you call this function with negative values for any of the parameters, or with Lmin or Lmax equal to 0.0, the writing of dmon boxes is disabled until the function is called again.

Note that the chromaticity parameters are quantized to the nearest multiple of 0.00002, while the luminance parameters are quantized to the nearest multiple of 0.0001. Moreover, chromaticities are clipped to the range 0 to 1.0 and luminances are clipped to fit within 32-bit unsigned integers after division by 0.0001.
