Synopsis

Returns 0, doing nothing, unless cbr_flushing returns true. In the CBR flushing mode, activated by the presence of a valid Scbr attribute with non-zero parameters, the codestream is flushed in a progressive manner subject to a low latency constraint, based upon a "leaky bucket" model, in which a constant bit-rate (CBR) channel drains the bucket at a constant rate, and new content is added to the bucket at defined "flush-set" boundaries. This function is normally called after the final flush call, to collect summary statistics concerning the performance of the CBR flushing mechanism.

Return Value

Number of flush events used to flush the codestream content. If the return value is 0, the other arguments are left untouched by the function; otherwise, they are used to return various statistics that may be of interest.

Arguments

bucket_max_bytes [float &]

Maximum allowed size of the leaky bucket, measured in bytes.

bucket_mean_bytes [float &]

Average size of the leaky bucket, measured in bytes.

inter_flush_bytes [float &]

Fixed number of bytes drained from the leaky bucket between flushes.

min_slope_threshold [kdu_uint16 &]

Minimum value of the distortion-length slope threshold selected over all flushes.

max_slope_threshold [kdu_uint16 &]

Maximum value of the distortion-length slope threshold selected over all flushes.

mean_slope_threshold [float &]

Average value of the distortion-length slope threshold over all flushes.

mean_square_slope_threshold [float &]

Average value of the squared distortion-length slope threshold over all flushes — you can combine this with mean_slope_threshold to calculate the slope variance, or the RMS slope variation.

num_fill_bytes [kdu_long &]

Total number of fill/spacer bytes inserted into all flush-sets in order to avoid underflow in the leaky bucket. This value does not include any underflow associated with the last flush-set, which is presumed to be handled at the application level by inserting fill/spacer bytes between the EOC marker of one codestream and the SOC marker of the next one.
