kdu_compositor_buf::get_rendering_status

Synopsis

This function is useful for applications that wish to display the content of composition buffers that are in the process of being rendered — e.g., after each return from the kdu_region_compositor::process function.

Upon return, num_rendered_rows will be equal to the number of initial rows of the buffer that can be considered to hold fully rendered content, while all rows from first_new_row to num_rendered_rows-1 can be considered to be newly rendered, since the function was last called with reset=true. It is often the case that first_new_row is equal to the value of num_rendered_rows from the last call with reset=true, but this is not necessarily the case. In particular, if the buffer has been refreshed or allocated since the last call to this function, first_new_row will be 0.

The function's return value is also important. It lets the caller know whether the imagery represented by the buffer has changed in some fundamental way, since the last call to this function with reset=true. Fundamental changes include changes in the buffer size or the true location (on the image) of the buffer surface, changes in the rendering scale or orientation, or changes in the image components, codestreams or compositing instructions (i.e., composited frame) associated with the buffer. Changes that are not considered fundamental are the removal or introduction of metadata overlays, and changes resulting from calls to kdu_region_compositor::refresh.

A false return means that there has been some fundamental change, in response to which the next attempt the application makes to display content from the buffer should involve displaying the entire buffer (or perhaps uploading it to a graphics texture for display), as opposed to displaying/updating just the rows delineated by first_new_row and num_rendered_rows. The content beyond num_rendered_rows will, in any case, at least hold erased background, so this is usually the least visually confusing thing to do.

Return Value

False if there has been some fundamental change in the imagery represented by the buffer, since the last call to this function with reset=true. True means that there has been no such fundmental change so that it would be appropriate to update a displayed version of the buffer just within the region identified by first_new_row and num_rendered_rows.

Arguments

num_rendered_rows [int &]

Used to return the number of initial buffer rows of the buffer that have been fully rendered so far. This value might be zero even if some region has been rendered, as returned via the process function, but the reverse will not be true.

Beyond these rendered rows, the buffer will always hold either erased background or meaningful content from previous rendering operations with the same fundamental interpretation (possibly with/without overlays or with lower quality data).

Rendering is complete once num_rendered_rows is equal to the height of the region returned by get_rendering_region.

first_new_row [int &]

Index of the first row of the buffer (0 is the index of its first row) that has been newly rendered, since the last call to this function that specified reset=true. The first_new_row is guaranteed to be no greater than num_rendered_rows; if equal, there are no new rendered rows.

reset [bool]

If true, the internal rendering status is reset so that the next call reports only changes that have occurred since this call. Otherwise, the next call will return exactly the same information as this one, if nothing has changed internally in the meantime.
