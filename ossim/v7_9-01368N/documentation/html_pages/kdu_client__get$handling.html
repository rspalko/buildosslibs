kdu_client::get_cache_file_handling

Java: Kdu_client.Get_cache_file_handling

int get_cache_file_handling( )

Java: public native int Get_cache_file_handling( )

Go to class description.

Synopsis

Returns the current cache file handling value. In practice, the returned value consists of zero or more of only the following two flags:

  • KDU_CLIENT_FILE_DELETE can be present if it was present in the most recent call to connect, open_with_cache_file or set_cache_file_handling, meaning that any compatible file in the cache directory passed to the first two of those functions either has been deleted or is about to be (on in the process of being) deleted.
  • KDU_CLIENT_FILE_SAVE can be present if it was present in the most recent call to connect, open_with_cache_file or set_cache_file_handling, meaning that a cache file will be written to the cache directory passed to the first two of these functions, once communication finishes or the close function is called.
