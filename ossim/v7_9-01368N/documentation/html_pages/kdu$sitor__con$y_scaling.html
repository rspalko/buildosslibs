Go to class description.

Synopsis

This function allows you to control how, or if, the underlying kdu_region_decompressor::set_true_scaling and kdu_region_decompressor::set_white_stretch functions are called when rendering is required.

Passing false for both arguments preserves (or reinstates) the default behaviour where kdu_region_decompressor::set_white_stretch is used to stretch low-precision source data out in to at least a full 8-bit dynamic range, but there is no other mechanism used to scale intensities where source and rendered output precisions differ. For unsigned source samples, this can lead to tiny departures from what an exact rendering should produce, but only in less common scenarios. The default behaviour for signed source data (not so common) is to map original 0 values to a mid-level grey rather than to black. This is usually a good starting point for visualising scientific data, since it allows both positive and negative departures from 0 to be visualised. Note that this default intensity mapping procedure was the only one available prior to KDU version 7.8.

For exact intensity mapping, you should pass true for both the true_zero and true_max arguments, but note that this may slightly increase the computational workload. To map signed source data in such a way that 0 maps to black rather than a mid-level grey, pass true_zero=true and (for the highest throughput) true_max=false.

Arguments

true_zero [bool]

true_max [bool]
